# 1. 题目介绍

- 题目：[湿地声景物种识别](https://www.kaggle.com/competitions/birdclef-2026)
- 数据集：由巴西潘塔纳尔湿地的 1000+ 台被动声学记录仪采集的连续音频构成
  - train_audio/：来自 xeno-canto 和 iNaturalist 的短时物种录音（ 32 kHz，ogg 格式 ）
  - train_soundscapes/：与测试集地点相近的 1 分钟声景录音，部分窗口已标注物种
  - train_soundscapes_labels.csv：训练声景的 5 秒窗口级标签，分号分隔的物种代码列表
  - test_soundscapes/：约 600 段 1 分钟隐藏测试录音
  - taxonomy.csv：物种分类信息，总共 234 类，包含鸟类、两栖类、哺乳类、昆虫、爬行类
- 特征
  - 音频全部重采样至 32 kHz
  - 训练数据包含两种来源：干净个体录音（ XC/iNat ）和现场声景标注（ 噪声更大 ）
  - 部分昆虫类别仅以声纹型态（ sonotype ）标注，未鉴定到具体种
- 目标变量
  - 对每一段 5 秒窗口预测 234 个物种的存在概率
  - 最终输出为 12 个窗口（ 共 1 分钟 ）的 row_id + 各物种概率
- 评估
  - Macro-Averaged ROC-AUC，跳过测试集中无正样本的类别

# 2. 数据预处理

- 音频标准化
  - 操作
    - 所有音频统一重采样至 32 kHz
      - 32 kHz 表示每秒对原始连续的声音波形进行 32,000 次采样
    - 多声道转单声道
      - 多声道音频包含两条或更多独立波形（ 如左声道、右声道 ），不同声道的声音可能有差异，从而营造立体声或环绕声效果
      - 因为 Perch 模型是在大规模单声道录音上训练的，因此其输入接口强制要求单声道
      - 方案
        - 取平均：最常用方案，将左右声道波形逐点相加取平均，保留全部声源的能量，不会丢失任何一侧的叫声
        - 选单道：只取左声道，简单但可能丢失仅在右声道出现的微弱叫声
        - 加权混合：较复杂，适用于已知声道质量差异的场景
- 窗口切分
  - 每段 1 分钟声景均匀切分为 12 个不重叠的 5 秒窗口
  - 以 5 秒窗口为单位进行预测
- Perch 特征提取
  - 使用 ONNX Runtime 批量运行 Perch v2 模型，输出
    - embeddings：(N, 1536)，深层声学特征
    - logits：(N, 234)，对应 234 个竞赛类别的初始得分
  - 获得高质量、可迁移的声学表征，作为后续所有模型的基础输入
- 缓存机制
  - 将训练集声景的 Perch 输出预先计算并保存为 .npz 文件，避免重复推理
- 物种代理映射
  - Perch 输出的 logits 中，Perch 词汇表未直接覆盖的物种（ 如部分昆虫、两栖类 ）最初输出为 0，需进行处理
  - 因为后续模型会进一步筛选，因此此步采取 “宁可错报，不可漏报” 的召回优先原则
  - 对于每个未被覆盖的物种，使用同属物种 logits 的最大值作为代理信号
    - 总共 234 类，其中直接映射成功 203 类，同属代理成功 3 类，完全无映射 28 类

# 3. 模型架构详解

## 3.1 基础编码器（ Perch v2 ）

- 模型来源：Google Bird Vocalization Classifier
- 输入：形状为 (batch, 160000) 的 5 秒音频波形（ 32kHz × 5s ）
- 输出：
  - embedding: (batch, 1536)，通用生物声学特征向量
  - label: (batch, 234)，原始分类 logits，原本有 14795 类，仅取与竞赛 234 类对应的子集
- 作用：提供强大的、预训练好的声学理解能力，全程冻结，不更新参数

## 3.2 时序建模分支（ LightProtoSSM + Cross-Attention ）

- 输入
  - 同一文件的 12 个窗口的 Perch 嵌入，shape 为 (batch_files, 12, 1536)
  - 对应的 Perch logits，shape 为 (batch_files, 12, 234)
  - 元信息，包括 地点 site 和 UTC 小时 hour 的嵌入向量
- 结构
  - 线性投影层 + 位置编码
  - 2 层选择性状态空间模型（ SSM ），分别沿正向和反向扫描时序
  - 层间加入 多头交叉注意力 增强特征交互
  - 可学习的类别原型向量与序列特征计算相似度，输出时序增强后的 logits
- 训练策略
  - 类别不平衡处理
    - 配置 pos_weight 参数，让少数类的预测错误在总损失中占更大比重
- 设计动机
  - 物种叫声在 1 分钟内存在时序相关性（ 如周期性鸣叫 ），SSM 能有效建模这种长程依赖，且计算量远小于 Transformer

## 3.3 逐类校准分支（ MLP 探针 ）

- 输入（ 针对每个物种独立构建 ）
  - PCA 投影向量（ 64 维 ）
    - 由 Perch 1536 维嵌入经标准化和主成分分析降维得到，代表该窗口的压缩声学表示
  - 原始 logit（ 1 维 ）
    - 当前窗口 Perch 对该物种的原始输出值
  - 时序统计量（ 5 维 ）
    - prev：同一文件内前一窗口（ 5 秒前 ）的 Perch logit
    - next：同一文件内后一窗口（ 5 秒后 ）的 Perch logit
    - mean：该文件 12 个窗口的 Perch logit 均值
    - max：该文件 12 个窗口的 Perch logit 最大值
    - std：该文件 12 个窗口的 Perch logit 标准差
- 结构
  - 小型全连接网络，隐藏层尺寸 (128, 64)，激活函数采用 ReLU，输出层为单一神经元
- 输出
  - 校准后的 logit（ 未经过 sigmoid ）
- 训练策略
  - 筛选条件：仅对训练集中正样本数 ≥ 5 的物种独立训练探针（ 避免样本过少导致过拟合 ）
  - 类别不平衡处理
    - 当一个类别（ 如稀有物种 ）的样本数量远少于其他类别时，模型会偏向于忽略稀有类别，导致这些类别的预测性能极差
    - 处理方案
      - 计算类别权重 class_weight = 1 / sqrt(freq)
      - 通过过采样正样本（ 重复 n_neg // n_pos 次 ）使正负样本数接近平衡
        - 例如通过把正样本复制 100+ 次，让正负样本比例更均衡，迫使 MLP 必须关注正样本的规律
  - 重要参数
    - 学习率 5e-4
    - L2 正则系数 α=0.005
    - 验证集比例 15%
    - 最大迭代 200 次（ 提交模式 ）或 500 次（ 训练模式 ）
    - 采用早停机制，n_iter_no_change 10 次（ 提交模式 ）或 20 次（ 训练模式 ）
- 设计动机
  - Perch 的原始 logit 来自线性分类器头，未能充分利用嵌入向量中的非线性判别信息，且逐窗口独立预测忽略了鸟类鸣声的时序连续性
  - MLP 探针通过以下两方面进行补偿
    - 声学精细校准：利用 PCA 保留的嵌入主成分学习物种特异的非线性校准函数，修正 Perch 的系统性偏差，如对稀有物种过于保守
    - 时序上下文注入：通过拼接前/后窗口分数及文件级统计量，让 MLP 隐式感知鸣叫事件的起止边界与持续时间，压制孤立噪声误检，提升连续鸣叫片段内的置信度一致性

## 3.4 二阶修正分支（ 残差 SSM ）

- 输入
  - Perch 嵌入（ 1536 维 ）
  - 第一阶段集成得分，即 ProtoSSM 与 MLP 的加权平均
- 结构：单层双向 SSM，输出加性修正项（ 输出层初始化为零 ）
- 训练目标：拟合真实标签与第一阶段预测概率之间的残差 (y - sigmoid(first_pass))
- 设计动机：第一阶段模型可能存在系统性偏差（ 例如对某些类别普遍高估或低估 ），残差 SSM 学习纠正这些偏差，进一步提升精度

## 3.5 元数据先验校正

- 方法：统计训练集中不同 site（ 地点 ）和 hour_utc（ UTC 小时 ）下的物种出现频率，构建先验概率表
- 应用：对于测试样本，根据其地点和小时查找对应先验概率，转换为 logit 偏移量后按权重 λ=0.4 加到原始 logits 上
- 设计动机：利用生态学常识——不同物种的活动具有时空规律（ 如夜行性、栖息地偏好 ），先验信息可有效提升罕见类的召回率

# 4. 推理流程

- 特征提取
  - 对测试声景运行 Perch ONNX 推理，得到 embedding 和初始 logits
- 第一阶段（ 融合 ）
  - 并行预测
    - 时序分支：该分支使用 原始 Perch logits（ 未加先验 ），运行 LightProtoSSM，采用 5 种循环移位 TTA（ shift = 0, ±1, ±2 ）并取平均，输出时序增强 logits
    - MLP 分支：先对原始 Perch logits 施加地点/小时频率先验偏移，再将调整后的 logits 与 Perch 嵌入输入逐类 MLP 探针（ 向量化批处理 ），输出校准后 logits
    - 先验分支：根据地点/小时查表，计算先验 logit 偏移
  - 将时序分支与 MLP 分支以 0.5 : 0.5 权重平均，输出第一阶段集成 logits
- 第二阶段（ 修正 ）
  - 将第一阶段的 logits 与 Perch 嵌入一同输入 ResidualSSM，得到修正项，乘上权重 0.30 后加回，得到最终 logits
- 概率转换
  - 应用逐类温度缩放：连续发声类（ 两栖、昆虫 ）温度略低（ T=0.95 ），事件性鸟类温度略高（ T=1.10 ）
  - Sigmoid 函数将 logits 映射为 \[0,1] 概率
- 后处理（ 按顺序执行 ）
  - 文件级置信度缩放：用文件内 Top-2 高分均值作为缩放因子（ power=0.4 ）
  - 排序感知缩放：用文件内最大分值作为额外缩放因子（ power=0.4 ）
  - 自适应 Delta 平滑：根据窗口置信度动态调整平滑强度，保护高置信峰值
  - 保序回归校准 + 最优阈值搜索：基于 OOF 预测拟合保序回归，并逐类搜索 F1 最优阈值，对概率进行非线性拉伸
- 生成提交文件
  - 直接输出 234 列连续概率值，格式与 sample_submission.csv 一致

# 5. 核心改进方向

## 5.1 移除随机性

- 每次拿到代码后，优化代码前，首先执行两遍，观察两次结果是否完全一致，从而检查代码中是否引入随机性
- 随机性会导致模型优化效果评估不准确，所以必须优先解决
- 原本代码方案中具有随机性，通过设置随机种子解决

  ```python
  def set_seed(seed=42):
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.use_deterministic_algorithms(True)

  set_seed(42)
  ```

## 5.2 模型结构优化

- 第一阶段优化
  - 添加 LightProtoSSM 变体，总共 3 种（ d_model=96, d_model=128, d_model=180） 三种变体，捕捉不同粒度的时序模式
- 第二阶段优化
  - 3 种深度残差模型 + 1 种贝叶斯收缩校正
    - 深度残差模型：包括 ResidualSSM、ResidualMLP、ResidualLinear
    - 贝叶斯收缩校正：能利用地点和小时的统计规律，温和地拉回第一阶段可能残留的系统偏差
  - 四者得分差别
    - ResidualSSM 0.928，ResidualMLP 0.929，ResidualLinear 0.928，贝叶斯收缩校正 0.928

## 5.3 MLP 参数优化

- 初始类别仅包括 PCA 投影向量（ 64 维 ）+ 原始 logit（ 1 维 ）+ 时序统计量（ 5 维 ）
- 尝试添加
  - ~~差分特征（ diff_mean/diff_prev/diff_next ）【 会导致劣化 】~~
    - diff_mean：base_col - mean_base，窗口与文件平均水平的偏离。正值表示该窗口比该文件整体更 “活跃”，可能对应物种的核心鸣叫时段
    - diff_prev base_col - prev_base，窗口与前一个窗口的差异。正值表示分数上升（ onset 检测 ），帮助模型定位声音的开始
