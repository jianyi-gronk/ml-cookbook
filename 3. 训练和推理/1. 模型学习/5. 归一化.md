## 1. 归一化

#### 1.1 背景

- **在神经网络的训练中，各层输入（ 即上一层的输出 ）的分布会随着参数更新而不断变化**，这种现象被称为 内部协变量偏移
- 这种分布变化可能导致以下问题
  - 梯度消失或爆炸：当激活值分布偏离合理范围时（ 如过大或过小 ），梯度可能变得极小或极大，阻碍优化过程
  - 训练不稳定：不同层的激活值分布差异会导致模型对初始参数和学习率敏感，增加训练难度
  - 收敛速度慢：分布不稳定的激活值可能导致梯度下降路径曲折，延长收敛时间

#### 1.2 基础介绍

- 和数据预处理的归一化类似，目的是 **调整数据分布，使模型训练更稳定**
- 区别在于使用的场景不同
  - 数据预处理的归一化
    - 模型训练前，对输入数据进行的标准化处理，统一特征尺度
  - 神经网络的归一化
    - 模型训练中（ **通常在 线性变化之后，激活函数之前** ），对每一层线性变换的结果进行标准化，以稳定中间层激活值的分布

## 2. 批归一化（ Batch Normalization，BN ）

- BN 通过 **对每个 batch 中的数据进行 标准化 处理，使得每层输入的 均值 接近 0，方差 接近 1，并且将标准化后的数据通过 缩放 和 平移，将数据重新映射到一个更合适的分布范围**
- 虽然实际上 是标准化 操作，但在论文中 Normalization 已成为泛指，涵盖了所有旨在 “调整数据分布以使训练更稳定”的技术

#### 2.1 详细步骤

- 对激活值 x 进行 **标准化**
  - 计算 batch 中数据的均值
    $$\mu = \frac{1}{m} \sum_{i=1}^m x_i$$
  - 计算 batch 中数据的方差
    $$\sigma^2 = \frac{1}{m} \sum_{i=1}^m (x_i - \mu)^2$$
  - 最终标准化
    $$\hat{x}_i = \frac{x_i - \mu}{\sqrt{\sigma^2 + \epsilon}}$$
  - $x_i$ 是 batch 中第 $i$ 个样本的线性变换结果，$m$ 是 batch size，$\epsilon$ 是一个小常数（ 通常为 $10^{-5}$，用于防止除零错误 ）
- **重构**，意思是对标准化后的值再进行一个仿射变换，即指一种线性变换（ 如缩放、旋转、剪切 ）加上平移的组合，这里是 缩放 + 平移
  $$y_i = \gamma \hat{x}_i + \beta$$
  - 其中，$\gamma$ 和 $\beta$ 是可学习的缩放和平移参数

#### 2.2 重构细节

- 核心是 **补偿归一化可能造成的信息损失，让网络自己决定数据的最佳分布，保持模型的容量**
- 重构的重要性
  - 通过引入可学习的 $\gamma$ 和 $\beta$，网络可以学习出最适合当前任务的分布
  - 如果 0 均值 1 方差对当前层最优，那 $\gamma$ 会趋近 1，$\beta$ 会趋近 0，等价于不做额外变换；如果需要其他分布，网络可以自由地调整 $\gamma$ 和 $\beta$ 来逼近任何均值和方差
  - 这样，BN 既享受了归一化带来的训练稳定性，又保留了网络的表征能力
  - 例如
    - 假设某一层的理想输出本应服从均值为 5、方差为 0.5 的分布（ 例如经过 sigmoid 激活函数后，可能需要偏大的输入值才能进入饱和区或非饱和区 ）
    - 如果强行把它变成均值为 0、方差为 1，那么后续层必须花费额外的学习成本去 “反向补偿” 这个变换，而且这种补偿是全局固定的，缺乏灵活性
    - 更严重的是，某些非线性激活函数（ 如 sgmoid、tanh ）在 0 附近处于近似线性的区域，这会削弱网络的非线性能力
- 如果没有重构，模型也可以正常训练，只不过会存在以下缺点
  - 收敛速度：没有重构，网络必须通过更深的层来缓慢地调整分布，训练会更慢
  - 性能上限：某些情况下，固定分布可能不是最优的，导致模型最终性能下降
  - 恒等映射：引入重构后，网络理论上可以学习一个恒等映射（ 当 $\gamma$ = $\sigma$, $\beta$ = $\mu$ 时，y 直接恢复为原始输入 x ），这意味着如果某一层不需要归一化，BN 层可以 “自我关闭”，不会对前向传播造成负面影响。如果没有重构，这个灵活性就丧失了

#### 2.3 不同场景

- 卷积神经网络
  - BN 常用于深度卷积神经网络（ 如 ResNet ）的卷积层之后，因卷积层的特征图具有空间一致性，BN 能有效标准化这些特征，减少内部协变量偏移
  - 空间一致性指卷积层特征图上所有像素都由同一卷积核生成，捕捉同一种特征（ 如边缘或纹理 ），因此数值分布（ 亮度或强度 ）在统计上相似
- 全连接层
  - BN 较少用于全连接层，因其输出缺乏特征图的空间结构，内部协变量偏移问题不显著，归一化收益较低
- 较大 batch size 场景
  - BN 依赖 batch 内样本的均值和方差计算，所以当 batch size 过小时，统计量不稳定，会导致归一化效果下降
- 生命周期
  - 在推理阶段（ 即模型用于预测或测试时 ），BN 使用训练过程中通过滑动平均累积的全局均值和方差，以确保对任意输入（ 包括单样本或不同 batch size ）进行一致的标准化

## 3. 层归一化（ Layer Normalization，LN ）

#### 3.1 与 LN 对比

- **BN 是对全部样本的一个特征进行归一化，而 LN 是对一个样本的全部特征进行归一化**，如果对人身体特征进行归一化，比如对身高，体重和年龄一起求一个均值方差，那确实看上去毫无作用，但在 NLP 领域却非常有效果
- 并且与 BN 不同，LN 不依赖 batch 内的统计量，而是对单个样本内的特征进行归一化，因此适用于 batch size 小或序列长度不一致的场景
- 和 LN 一样，可以通过可学习的缩放参数 $\gamma$ 和平移参数 $\beta$，LN 调整标准化后的数据，保留网络的表达能力

#### 3.2 在 NLP 领域的优势

- 在 NLP 中，n 个特征都可能表示不同的词，如果仍然采用 BatchNormalization 的话，对第一个词进行操作，很显然意义就不是非常大了，因为任何一个词都可以放在第一个位置，而且很多时候词序对于我们对于句子的影响没那么大，而此时我们对 n 个词进行 LN 等操作可以很好地有效捕捉样本（ 如句子 ）的整体分布特性，而不受词序或 batch 大小的影响

#### 3.3 不同场景

- NLP 任务
  - LN 常用于自然语言处理中的序列模型（ 如 RNN、Transformer ），例如，在 Transformer 中，LN 应用于每个样本的隐藏状态（ 如注意力层的输出 ），稳定句子的特征分布，而不关心词序或 batch 中其他样本
- 小 batch size 或单样本场景
  - LN 不依赖 batch 统计量，适合 batch size 为 1 或动态变化的场景
- 生命周期
  - LN 在训练和推理阶段都使用样本内统计量，无需存储全局均值和方差，计算更简单

## 4. 实例归一化（ Instance Normalization，IN ）

#### 4.1 背景

- **BN 在 图像生成任务 中表现不佳**，例如 风格迁移 通常需要每个样本独立处理，保持实例的特定风格信息，而 BN 会混用 batch 内多个样本的统计量，导致生成图像丢失个体细节
- Ulyanov 等人在《Instance Normalization: The Missing Ingredient for Fast Stylization》中提出 IN，对每个样本的每个通道单独进行归一化，以保留每个实例的独立特征

#### 4.2 计算方式

- 统计维度：对每个样本 $i$、每个通道 $c$，仅在当前样本当前通道的 **空间维度**（ 高度 $H$ 和宽度 $W$ ）上计算均值和方差
- 公式
  - 对激活值 $x_{i,c,h,w}$（ $i$ 为样本索引，$c$ 为通道索引，$h,w$ 为空间位置 ），计算
    $$
    \mu_{i,c} = \frac{1}{HW} \sum_{h=1}^{H} \sum_{w=1}^{W} x_{i,c,h,w}
    $$
    $$
    \sigma_{i,c}^2 = \frac{1}{HW} \sum_{h=1}^{H} \sum_{w=1}^{W} (x_{i,c,h,w} - \mu_{i,c})^2
    $$
  - 标准化
    $$
    \hat{x}_{i,c,h,w} = \frac{x_{i,c,h,w} - \mu_{i,c}}{\sqrt{\sigma_{i,c}^2 + \epsilon}}
    $$
  - 重构
    $$
    y_{i,c,h,w} = \gamma_c \hat{x}_{i,c,h,w} + \beta_c
    $$
    其中 $\gamma_c, \beta_c$ 是每个通道可学习的参数（ 与 BN 相同 ）

#### 4.3 不同场景

- 图像风格迁移/生成任务：IN 能有效去除对比度等低级统计信息，使模型更专注于内容结构，因此成为风格迁移、GAN 等生成模型的标准组件
- 图像分割/检测：在某些任务中也可用，但不如 GN 或 BN 普遍
- 生命周期：训练和推理阶段均使用当前样本当前通道的统计量，无需存储全局统计量

## 5. 组归一化（ Group Normalization, GN ）

#### 5.1 背景

- BN 对小 batch size 敏感：当 batch size 很小时（ 如目标检测、视频分割任务中 batch 通常为 1 或 2 ），BN 计算的统计量噪声很大，导致模型性能严重下降
- GN 是 LN 和 IN 的折衷
  - LN 对整个样本所有通道归一化，可能破坏通道间的独立性
  - IN 对每个通道单独归一化，可能丢失通道间的相关性
  - 而 GN 将通道分组，在组内归一化，既避免了对 batch 的依赖，又保留了组内通道的联合信息

#### 5.2 计算方式

- 统计维度：对每个样本 $i$、每个组 $g$（ 包含 $C/G$ 个通道 ），仅在当前样本当前组的 **所有通道和空间维度** 上计算均值和方差
- 公式
  - 将通道分组后，对第 $i$ 个样本、第 $g$ 组（ 包含 $C/G$ 个通道 ）的激活值 $x_{i,g,c,h,w}$（ $c$ 为组内通道索引 ），计算
    $$
    \mu_{i,g} = \frac{1}{(C/G) \cdot H \cdot W} \sum_{c=1}^{C/G} \sum_{h=1}^{H} \sum_{w=1}^{W} x_{i,g,c,h,w}
    $$
    $$
    \sigma_{i,g}^2 = \frac{1}{(C/G) \cdot H \cdot W} \sum_{c=1}^{C/G} \sum_{h=1}^{H} \sum_{w=1}^{W} (x_{i,g,c,h,w} - \mu_{i,g})^2
    $$
  - 标准化
    $$
    \hat{x}_{i,g,c,h,w} = \frac{x_{i,g,c,h,w} - \mu_{i,g}}{\sqrt{\sigma_{i,g}^2 + \epsilon}}
    $$
  - 重构
    $$
    y_{i,g,c,h,w} = \gamma_{g,c} \hat{x}_{i,g,c,h,w} + \beta_{g,c}
    $$
    其中 \(\gamma*{g,c}, \beta*{g,c}\) 是每个组内通道可学习的参数（ 也可简化为每个通道独立 ）

#### 5.3 不同场景

- 小 batch size 的视觉任务：如目标检测（ Mask R-CNN ）、语义分割（ DeepLab ）等，batch size 常为 1 或 2，GN 效果优于 BN
- 视频理解：处理视频帧时 batch 受限，GN 能稳定训练
- 与 LN 的对比：LN 可视为组数 $G=1$ 的特例，IN 可视为组数 $G=C$（ 每组一个通道 ）的特例。GN 通过调节 $G$ 平衡了 LN 和 IN 的特性
